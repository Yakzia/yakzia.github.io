Mensaje de advertencia al intentar guardar los cambios de diseño en SQL Server Management Studio

#Técnico — miércoles, 25 de marzo de 2020

Trabajando con Microsoft SQL Server Management Studio (también conocido como MSSQLMS o SSMS) quizás te encuentres con un mensaje similar al siguiente:

Este mensaje suele presentarse cuando se intenta guardar una tabla después de cambiar su diseño (ya sea que agreguemos, eliminemos o renombremos campos, que establezcamos un campo como clave principal, entre otros cambios) y se requiere que se elimine la tabla y se vuelva a crear.

Una opción para hacer que se guarden los cambios es abrir el menú Tools y a continuación elegir Options.

En el cuadro de diálogo Options expandimos la categoría Designers y seleccionamos Table and Database Designers. En la sección derecha encontraremos la opción Prevent saving changes that require table re-creation la cual será necesario desactivar.

Con esto, será posible guardar los cambios en el diseño de las tablas y no se mostrará la advertencia.
